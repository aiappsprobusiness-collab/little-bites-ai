# Навигация экрана «План» (`/meal-plan`)

## Выбранный день недели (rolling 7 дней)

**Источник истины:** query-параметр **`date`** в URL — локальная дата в формате **`YYYY-MM-DD`**, должна совпадать с одним из семи дней текущего rolling-окна (`getRolling7Dates`).

- **Premium (и прочие с доступом к неделе):** при тапе по вкладке дня в шапке плана вызывается `setSearchParams` с **`replace: true`** — в URL выставляется `?date=…` без лишних записей в истории. При возврате с экрана рецепта (`history.back`) в стеке остаётся та же строка запроса, и день восстанавливается при первом рендере (через `useMemo`, без промежуточного «прыжка» на первый день).
- **Free:** день зафиксирован на «сегодня»; если в URL нет подходящего `date` или он не совпадает с сегодняшним днём в диапазоне, клиент синхронизирует query с сегодняшней датой (`replace`), чтобы поведение шаринга и прямых ссылок оставалось предсказуемым.

Имя параметра и вспомогательный путь возврата из рецепта: `src/utils/mealPlanNavigation.ts` (`MEAL_PLAN_DATE_QUERY_PARAM`, `mealPlanPathWithOptionalDate`).

## Возврат из карточки рецепта

Открытие рецепта с плана задаёт в `location.state` поле **`plannedDate`** (тот же `YYYY-MM-DD`). Кнопка «назад» / «К плану питания» на `RecipePage` ведёт на **`mealPlanPathWithOptionalDate(plannedDate)`**, то есть на `/meal-plan?date=…`, а не на «голый» `/meal-plan`.

## Связь с другими входами

- Редирект после создания члена семьи уже использует `?memberId=&date=` — см. `getPlanUrlForMember` в `src/services/planFill.ts` (тот же параметр `date`).
- Нижняя вкладка «План» по-прежнему ведёт на `/meal-plan` без query: для Premium это соответствует **первому дню** rolling-окна, если не задан `date` (как и раньше при размонтировании страницы).

## Реализация

Основная логика: `src/pages/MealPlanPage.tsx` (вычисление `selectedDay` из `searchParams` + `selectPlanDayIndex`).
