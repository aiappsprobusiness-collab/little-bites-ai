# План (вкладка): тихий hero и иерархия CTA (март 2026)

Документ фиксирует **намеренные UX/UI-решения** после локальной чистки шума на экране Плана. Логика генерации, paywall и API **не менялись**, кроме переноса действий в меню.

## Что изменили в UI

1. **Верхняя карточка (hero)**  
   - Фон: `bg-card/70`, без жёсткой рамки: `border-0 shadow-none ring-1 ring-border/20` (меньше «коробки», больше воздуха).  
   - Разделитель блока действий: очень лёгкий `border-t border-border/15`.  
   - Бейдж подписки: компактнее (`rounded-full`, приглушённые тона).

2. **Подзаголовок про профиль**  
   - Строки «Учитываем все особенности профиля» и блок семейного режима с иконкой **убраны из hero**.  
   - Тот же смысл доступен из меню «Ещё» → **«Как учитывается профиль»** (нижний sheet с `PlanProfileHelpBody`).

3. **Цели подбора**  
   - В hero: **только значение** (например «Баланс») + стрелка; подписи «Цель подбора» в UI нет (остаётся в `aria-label` и в заголовке sheet).  
   - Блок без «карточности»: прозрачный фон, без рамки, лёгкий hover по тексту (`PlanGoalCompactSheet`).  
   - Полный список чипсов — в **нижнем sheet** (режим `density="sheet"`).  
   - Сохранены: один выбор, сброс по повторному тапу, Free + 🔒 + paywall на премиум-целях в sheet.

4. **Главные действия (иерархия)**  
   Порядок сверху вниз, **одна колонка**:
   1. **Primary:** «Собрать день» — полная ширина, градиент primary.  
   2. **Secondary:** «Поделиться днём» — `Button variant="outline"`, полная ширина, мягкая рамка.  
   3. **Tertiary / premium teaser:** «Собрать неделю» — **не** `Button`, обычный текстовый контрол (`<button>`): мелкий шрифт, `text-muted-foreground`, hover underline; **без** заливки, тени и иконки Sparkles.  
   - **Free:** рядом 🔒 и метка **Premium** (апселл); по клику — `FF_WEEK_PAYWALL_PREVIEW` → preview sheet, иначе paywall с прежним текстом.  
   - **Premium / Trial:** только текст «Собрать неделю»; по клику — прежний `runPoolUpgrade` на неделю.  
   - **Обоснование:** неделя остаётся discoverable и ведёт на апселл, но визуально не конкурирует с днём.

5. **Шаринг**  
   - В hero остаётся только **«Поделиться днём»** (outline full width; «днём»).  
   - **«Поделиться неделей»** убрана из hero; для **Premium/Trial** добавлена в меню «Ещё».  
   - `shareWeekPlan` и `createSharedPlan` **не трогались**; для Free пункт «Поделиться неделей» в меню **не показываем** (раньше на главной он вёл на paywall — сценарий недели по-прежнему доступен через «Собрать неделю»).

6. **Карточки блюд на плане**  
   - Цели питания под превью: максимум **1** тег, стиль **`quiet`** (`NutritionGoalsChips` + пропсы в `RecipeCard` / `MealCard` compact).

## Ручная проверка (dev)

- [ ] Hero: дата, профиль; цель — одна строка-значение без заголовка «Цель подбора».  
- [ ] Колонка действий: «Собрать день» → «Поделиться днём» (outline) → текстовая «Собрать неделю» (у Free: 🔒 + Premium).  
- [ ] Free: третья строка открывает paywall / week preview; в меню нет «Поделиться неделей».  
- [ ] Premium/Trial: третья строка запускает сбор недели; в меню «Поделиться неделей» как раньше.  
- [ ] Цель: тап по значению → sheet, выбор цели, paywall на заблокированных (Free).  
- [ ] Меню → «Как учитывается профиль»: корректный текст для семьи и одного члена.  
- [ ] Карточка слота: не больше одного чипа цели, визуально спокойнее.  
- [ ] Баннер после первого плана («План готов!»): кнопка «Поделиться меню на день» на месте.

## Сознательно не трогали

- Другие вкладки (Чат, Избранное, Профиль и т.д.).  
- `SharedPlanPage`, лендинги, аналитика (кроме косвенного смещения UI для `share_week` — события на лендинге не менялись).  
- Edge `generate-plan`, пул, replace/delete слотов.  
- Горизонтальные чипсы дней недели под hero.  
- Пустые состояния и дублирующие «Собрать день» внизу списка.
