# План (вкладка): тихий hero и иерархия CTA (март 2026)

Документ фиксирует **намеренные UX/UI-решения** после локальной чистки шума на экране Плана. **Обновление (март 2026):** логика `generate-plan` и фронт перестали использовать выбор цели питания в Плане — см. §3 и `docs/dev/meal-plan-nutrition-goals-2026-03-progress.md`.

## Что изменили в UI

1. **Верхняя карточка (hero)**  
   - Фон: `bg-card/70`, без жёсткой рамки: `border-0 shadow-none ring-1 ring-border/20` (меньше «коробки», больше воздуха).  
   - Разделитель блока действий: очень лёгкий `border-t border-border/15`.  
   - Бейдж подписки: компактнее (`rounded-full`, приглушённые тона).

1a. **Дата в hero**  
   - Одна строка через `formatDayHeader`: **«Воскресенье, 22 марта»** (`ru-RU`, месяц родительный падеж из `MONTHS_GENITIVE`).  
   - Слова **«Сегодня»** нет; вторая строка только с датой **убрана**.

2. **Подзаголовок про профиль**  
   - Строки «Учитываем все особенности профиля» и блок семейного режима с иконкой **убраны из hero**.  
   - Тот же смысл доступен из меню «Ещё» → **«Как учитывается профиль»** (нижний sheet с `PlanProfileHelpBody`).

3. **Цели подбора (обновление март 2026)**  
   - Селектор цели питания под датой **убран**: nutrition goals не задают приоритет подбора плана; теги по-прежнему на карточках блюд и в рецепте.  
   - Компонент `PlanGoalChipsRow` / `PlanGoalCompactSheet` и утилита `planGoalSelect` **удалены**; `usePlanGenerationJob` больше не шлёт `selected_goal` в `generate-plan`.

4. **Главные действия в hero (иерархия)**  
   Порядок сверху вниз, **одна колонка**:
   1. **Primary:** «Собрать день» — полная ширина, градиент primary.  
   2. **Tertiary / premium teaser:** «Собрать неделю» — **не** `Button`, обычный текстовый контрол (`<button>`): мелкий шрифт, `text-muted-foreground`, hover underline; **без** заливки, тени и иконки Sparkles.  
   - **Free:** рядом 🔒 и метка **Premium** (апселл); по клику — `FF_WEEK_PAYWALL_PREVIEW` → preview sheet, иначе paywall с прежним текстом.  
   - **Premium / Trial:** только текст «Собрать неделю»; по клику — прежний `runPoolUpgrade` на неделю.  
   - **Обоснование:** неделя остаётся discoverable и ведёт на апселл, но визуально не конкурирует с днём.

4a. **Под списком блюд**  
   - **Secondary:** «Собрать список продуктов» — outline, заметнее.  
   - **Ghost:** «Отправить меню» — только если в выбранном дне есть блюда (`recipe_id`); логика `shareDayPlan` без изменений.

5. **Шаринг**  
   - **«Отправить меню»** (день) — под карточками приёмов пищи, после списка продуктов.  
   - **«Отправить меню на неделю»** — в меню «Ещё» для **Premium/Trial** (подпись обновлена с «Поделиться неделей»).  
   - `shareWeekPlan` и `createSharedPlan` **не менялись**; для Free пункт недели в меню **не показываем**.

6. **Карточки блюд на плане**  
   - Цели питания под превью: максимум **1** тег, стиль **`quiet`** (`NutritionGoalsChips` + пропсы в `RecipeCard` / `MealCard` compact).

## Ручная проверка (dev)

- [ ] Заголовок: **одна строка** «День недели, D месяца»; для сегодняшнего дня нет префикса «Сегодня».  
- [ ] Под датой: **профиль и цель рядом** (`justify-start gap-3`), у цели **ChevronDown**.  
- [ ] Тап по цели открывает **диалог** как у профиля; выбор строки, 🔒 + paywall для Free; повторный тап по выбранной цели сбрасывает в «баланс» (`onChange(null)`).  
- [ ] Генерация плана: обе пилюли неактивны.  
- [ ] Hero: «Собрать день» → текстовая «Собрать неделю» (у Free: 🔒 + Premium).  
- [ ] Под блюдами: «Собрать список продуктов» → ghost «Отправить меню» (только если день не пустой).  
- [ ] Free: вторая строка hero открывает paywall / week preview; в меню нет «Отправить меню на неделю».  
- [ ] Premium/Trial: вторая строка hero запускает сбор недели; в меню «Отправить меню на неделю».  
- [ ] Цель: тап по значению → sheet, выбор цели, paywall на заблокированных (Free).  
- [ ] Меню → «Как учитывается профиль»: корректный текст для семьи и одного члена.  
- [ ] Карточка слота: не больше одного чипа цели, визуально спокойнее.  
- [ ] Баннер после первого плана («План готов!»): кнопка «Отправить меню» на месте.

## 7. Прикорм (&lt;12 мес) — март 2026 (точечная доработка)

**Файлы:** `src/pages/MealPlanPage.tsx`, `src/index.css` (при необходимости), `src/utils/introducedProducts.ts` (`getInfantPrimaryProductSummaryParts`), при необходимости тесты в `introducedProducts.test.ts`.

**Изменения UX:**

- Скролл прикорма: у `html` — `scrollbar-gutter: auto` вместо `stable` (убирает постоянную полосу-гаттер у края вьюпорта); на экране прикорма — `scrollbar-none` + `!overflow-y-hidden` на `main` и `scrollbar-none` на внутреннем `plan-page-bg`, чтобы полоса скролла не была видна, прокрутка остаётся на внутреннем контейнере.
- CTA под primary: **«Ввести &lt;продукт&gt; →»** вместо формулировки «добавить в введённые».
- Под чипом ребёнка — **единый info-блок** (справочный мелкий текст + при 4–5 мес предупреждение про врача); CTA ниже блока.
- Hero прикорма: заголовок + дата + бейдж/меню — **отдельная верхняя строка**; блок с чипом и текстом на **полную ширину карточки** (убрано сжатие основного текста из‑за соседней колонки с меню в одном flex-ряду).
- **«Помощь маме»** и **«Уже введённые продукты»** — не на всю ширину: `max-w` ~половина контейнера на `sm+`, на узком экране до ~100% с разумным пределом в `rem`.
- Заголовок прикорма: **«План прикорма на сегодня»**; над primary — **`getInfantPrimaryProductSummaryParts`** («Новый продукт: …», при необходимости «Знакомый продукт: …»); над secondary — **«Уже знакомое блюдо»**; **«Помощь маме»** и **«Уже введённые продукты»** визуально вторичные (outline / ghost).

**Ручная проверка:** порядок блоков в hero; длина строк основного абзаца на широком телефоне; ширина CTA; читаемость секции над первой карточкой; второй слот; профиль 12+ и «Семья» без регрессий.

Каноническое описание домена: **`docs/architecture/PLAN_MENU_PROFILE_AND_RECIPE_SELECTION.md`** (§5.4, §10).
