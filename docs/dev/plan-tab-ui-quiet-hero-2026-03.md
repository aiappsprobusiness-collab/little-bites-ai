# План (вкладка): тихий hero и иерархия CTA (март 2026)

Документ фиксирует **намеренные UX/UI-решения** после локальной чистки шума на экране Плана. Логика генерации, paywall и API **не менялись**, кроме переноса действий в меню.

## Что изменили в UI

1. **Верхняя карточка (hero)**  
   - Фон: `bg-card/85`, рамка `border-border/40`, лёгкая `shadow-sm` вместо сочного `primary-light` и тяжёлой тени.  
   - Бейдж подписки: компактнее (`rounded-full`, приглушённые тона).

2. **Подзаголовок про профиль**  
   - Строки «Учитываем все особенности профиля» и блок семейного режима с иконкой **убраны из hero**.  
   - Тот же смысл доступен из меню «Ещё» → **«Как учитывается профиль»** (нижний sheet с `PlanProfileHelpBody`).

3. **Цели подбора**  
   - В hero: одна компактная строка **«Цель подбора» + текущее значение** (`PlanGoalCompactSheet`).  
   - Полный список чипсов — в **нижнем sheet** (все цели, режим `density="sheet"` с более спокойными неактивными чипсами).  
   - Сохранены: один выбор, сброс по повторному тапу, Free + 🔒 + paywall на премиум-целях.

4. **Главные действия**  
   - **Одна визуально доминирующая CTA:** «Собрать день» на всю ширину (primary, без соседа в одной сетке).  
   - **«Собрать неделю»:** `variant="ghost"`, меньшая иконка Sparkles, без второго «primary» блока; для Free — тот же paywall / preview, что и раньше.  
   - **Обоснование:** недельный сценарий остаётся на виду рядом с шарингом дня, но не конкурирует с дневной генерацией по весу кнопки.

5. **Шаринг**  
   - В hero остаётся только **«Поделиться днём»** (outline, скорректирована орфография: «днём»).  
   - **«Поделиться неделей»** убрана из hero; для **Premium/Trial** добавлена в меню «Ещё».  
   - `shareWeekPlan` и `createSharedPlan` **не трогались**; для Free пункт «Поделиться неделей» в меню **не показываем** (раньше на главной он вёл на paywall — сценарий недели по-прежнему доступен через «Собрать неделю»).

6. **Карточки блюд на плане**  
   - Цели питания под превью: максимум **1** тег, стиль **`quiet`** (`NutritionGoalsChips` + пропсы в `RecipeCard` / `MealCard` compact).

## Ручная проверка (dev)

- [ ] Hero: дата, профиль, одна primary «Собрать день», рядом вторым рядом — «Поделиться днём» + ghost «Собрать неделю».  
- [ ] Free: «Собрать неделю» открывает paywall / preview; в меню нет «Поделиться неделей».  
- [ ] Premium/Trial: «Собрать неделю» заполняет неделю; в меню «Поделиться неделей» работает как раньше.  
- [ ] Цель: тап по компактной строке → sheet, выбор цели, paywall на заблокированных (Free).  
- [ ] Меню → «Как учитывается профиль»: корректный текст для семьи и одного члена.  
- [ ] Карточка слота: не больше одного чипа цели, визуально спокойнее.  
- [ ] Баннер после первого плана («План готов!»): кнопка «Поделиться меню на день» на месте.

## Сознательно не трогали

- Другие вкладки (Чат, Избранное, Профиль и т.д.).  
- `SharedPlanPage`, лендинги, аналитика (кроме косвенного смещения UI для `share_week` — события на лендинге не менялись).  
- Edge `generate-plan`, пул, replace/delete слотов.  
- Горизонтальные чипсы дней недели под hero.  
- Пустые состояния и дублирующие «Собрать день» внизу списка.
