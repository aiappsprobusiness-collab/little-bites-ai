# Локальный журнал изменений VK-воронки

В репозитории **не хранится** рабочий журнал: его ведут в файле в корне проекта **`VK_FUNNEL_LOCAL_CHANGELOG.md`**.

- Файл перечислен в **`.gitignore`**, в прод и в git **не попадает**.
- Создайте `VK_FUNNEL_LOCAL_CHANGELOG.md` вручную в корне (рядом с `package.json`), если ещё нет.
- Ссылка на путь дублируется в **`.env.example`** (закоммиченном шаблоне), чтобы не забывать, что журнал — только локально.

## Формат записи

Каждое событие — блок с **датой и временем по Москве (МСК)** и краткими полями:

1. **Что сделано** — суть правки, при необходимости пути к файлам.
2. **Зачем** — бизнес-цель / контекст.
3. **Когда** — `ГГГГ-ММ-ДД ЧЧ:ММ:МС` (МСК).

Пример:

```markdown
## 2026-04-26 00:17:32 (МСК)

**Что сделано:** …

**Зачем:** …
```

Правки продуктового поведения воронки по-прежнему фиксируются в `docs/architecture/vk-funnel.md`; этот журнал — для **дополнительного** трекинга и заметок у разработчика, без обязательного PR-описания.
