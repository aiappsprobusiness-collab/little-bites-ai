# Вкладка «Избранное»: спокойный UI коллекции (март 2026)

## Цель

Собрать экран **без смены продуктовой логики**: меньше визуального шума в шапке, тише фильтр и карточки, явная иерархия «коллекция рецептов», премиальный тон.

## Что изменено

### Шапка экрана (`FavoritesPage.tsx`)

- **Табы «Избранное / Мои рецепты»** — единственный крупный переключатель сверху: сегмент в `rounded-xl` контейнере с `bg-muted/35`, активный сегмент — `bg-card` + лёгкая тень (без отдельных «пилюль» с оливковой обводкой).
- **Селектор профиля** — на следующей строке, слева, вариант `MemberSelectorButton` **`variant="light"`** (вторичный, без тяжёлой primary-pill заливки).
- **«Покупки»** — не в одном ряду с табами: **справа на второй строке**, текст `text-xs`, иконка корзины приглушена; для Free по-прежнему иконка замка и тот же `openShoppingList` → paywall или вкладка списка.
- Экран **«Список покупок»** по-прежнему открывается табом внутри страницы; на нём остаётся ссылка **«К избранному и рецептам»** (слегка укрупнён шрифт до `text-sm` для читаемости).

### Подпись над списком

- При непустом списке: короткая строка **`Сохранённые рецепты`** (избранное) или **`Свои рецепты в коллекции`** (мои рецепты), `text-xs text-muted-foreground/80` — задаёт смысл экрана без дублирования табов.

### Фильтр по ингредиентам (`IngredientFilterBar.tsx`)

- **Поле ввода:** шире на мобиле (`flex-1 min-w-0`), `rounded-xl`, `bg-muted/20`, бордер `border-border/50`, кольцо фокуса слабее (`ring-1`), иконка поиска мельче и тише.
- **Плейсхолдер по умолчанию:** `Найти по ингредиенту…` (поле по-прежнему добавляет **токены ингредиентов**, а не полнотекстовый поиск по названию рецепта; формулировка из ТЗ «Поиск рецептов» сознательно **не** использована, чтобы не вводить в заблуждение).
- **«Есть / Нет»:** логика `include` / `exclude` **без изменений**; визуально перенесены в **микро-сегмент** `rounded-full` с нейтральным фоном, активный режим — белая «таблетка» с тенью, без заливки `bg-primary` на весь переключатель (раньше переключатель конкурировал с табами и полем).
- Чипсы выбранных ингредиентов: нейтральные `bg-muted/40`, без яркого primary-контура.

### Карточки списка (`FavoriteCard`, `MyRecipeCard`, `RecipeCard`, `RecipeHeader`, `RecipeNutritionHeader`)

- Для превью коллекции добавлен режим **`previewPresentation="collection"`** в `RecipeCard` (используется **только** из избранного/моих рецептов; **MealCard** не трогаем).
- Фон карточки: нейтральный `bg-card`, бордер `border-border/60`, очень лёгкая тень вместо оливкового `bg-primary/[0.06]`.
- Заголовок блюда в компактной шапке: чуть крупнее и **`font-semibold`** (`compactCollection` в `RecipeHeader`).
- Мета (тип приёма, время, ккал, БЖУ): **`nutritionTone="quiet"`** — бейдж приёма без сплошного primary, ккал без тяжёлого чипа, макросы мельче и с меньшим контрастом.
- Чипсы целей питания: максимум **2** (`nutritionGoalsMaxVisible={2}`), стиль тише за счёт существующего `quiet` в `NutritionGoalsChips`.
- Колонка действий справа: кнопки **`h-9 w-9 rounded-xl`**, спокойные бордеры; сердце — на приглушённом фоне `bg-muted/30`, без «кричащей» primary-заливки всей кнопки.
- Отступы тела карточки и колонки иконок слегка увеличены; на странице список карточек **`space-y-4`**.

## Что проверить вручную (dev)

1. Переключение **Избранное / Мои рецепты** и сохранение deep-link `state.tab`.
2. **Селектор профиля** (Premium и Free / locked).
3. **Покупки:** Premium → открывается список; Free → paywall с прежним сообщением.
4. Фильтр: ввод ингредиента, Enter, чипсы, **Есть/Нет** (рецепты фильтруются как раньше), сброс.
5. Тап по карточке, **сердце** (удаление из избранного), **календарь** (в план), **карандаш** у своих рецептов.
6. Вкладка списка покупок и возврат **«К избранному и рецептам»**.
7. Карточки на **Плане** (MealCard): визуал превью **не** должен совпасть с новым «collection» (остаётся прежний `preview` без `previewPresentation`).

## Сознательно не трогали

- Хуки данных (`useFavorites`, `useRecipeIdsByIngredients`, `useMyRecipes`), навигация в `App`, `BottomNavigation`, `ShoppingListView` (кроме входа со страницы), `MealCard` / план, промпты, Edge, схема БД.
- Пустые состояния и тексты CTA в empty state (кроме общей шапки страницы).
