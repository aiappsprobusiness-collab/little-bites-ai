# Фильтр списка покупок по рецептам (2026-03)

## Root cause

- Два несогласованных флага: **`draftAllRecipes`** и **`draftRecipeIds`**.
- При `draftAllRecipes === true` строки рецептов показывались как `!draftAllRecipes && draftRecipeIds.has(id)` → **всегда выключены**, хотя master был включён.
- Строки рецептов были **`disabled` при «Все рецепты»**, поэтому нельзя было снять отдельные рецепты без лишнего шага.
- При «Применить» условие `draftAllRecipes || draftRecipeIds.size === 0` трактовало **пустой** `draftRecipeIds` как «показать всё», в том числе после снятия master — логика и визуал расходились.

## Новая модель

| Слой | Состояние |
|------|-----------|
| Applied (после «Применить») | `selectedRecipeIds`: `size === 0` → без фильтра по рецептам; иначе — только позиции с пересечением по источникам |
| Черновик в sheet | `draftRecipeIds`: явный набор выбранных id из `uniqueRecipes` |
| Master «Все рецепты» | Производный: все id выбраны → `checked`; часть → `indeterminate` (через `ref.indeterminate`); ноль → `unchecked` |

Гидрация при **открытии** sheet (переход closed → open): если applied пустой — `draftRecipeIds = all ids`; иначе копия `selectedRecipeIds`.

## Как синхронизируются master и строки

- Снять **«Все рецепты»** → `draftRecipeIds = new Set()` (все конкретные сняты).
- Включить **«Все рецепты»** → `draftRecipeIds =` все id из `uniqueRecipes`.
- Переключить отдельный рецепт → add/remove в `draftRecipeIds`; master пересчитывается из `draftRecipeIds` и `uniqueRecipes`.

## Изменённые файлы

- `src/components/favorites/ShoppingListView.tsx` — state, hydrate, master/indeterminate, apply, отключение «Применить» при нуле выбранных при наличии рецептов.
- `docs/architecture/shopping_list_product_model.md` — раздел «Фильтр по рецептам», пункт чеклиста 9.

## Что проверить руками

1. Список с несколькими рецептами → открыть фильтр → **все** чекбоксы рецептов отмечены, master включён.
2. Снять один рецепт — список остаётся с фильтром; master **indeterminate** (или снят, если один рецепт).
3. Снять master — все конкретные сняты; «Применить» **неактивна**; выбрать 1–2 рецепта — «Применить» активна, список покупок фильтруется.
4. Включить все рецепты вручную по одному — master становится **включённым**; «Применить» сбрасывает фильтр (пустой `selectedRecipeIds`).
5. Категория, поиск по рецепту в sheet, «Только некупленные», сброс фильтров — без регрессий.

## Сознательно не трогалось

- Остальные фильтры списка (кроме синхронизации UI черновика рецептов).
- `useShoppingList` и API.
