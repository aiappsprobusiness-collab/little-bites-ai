# Поле ввода чата (UI) — источник истины

Документ фиксирует поведение и реализацию **нижней панели ввода** на экране чата (вкладка «Чат»), чтобы при правках UX не ломать авто-рост, клавиатуру и выравнивание кнопки отправки.

---

## Где в коде

| Что | Файл |
|-----|------|
| Визуал и логика поля + кнопка «Отправить» | `src/components/chat/ChatInputBar.tsx` |
| Состояние текста, ref на textarea, отправка по Enter | `src/pages/ChatPage.tsx` |

Отдельный сценарий — нижний sheet «Написать сообщение» (`src/components/chat/ChatInputPanel.tsx`): это **не** та же панель, что у основного чата; при изменении UX там правки документируются отдельно при необходимости.

---

## Разметка и дизайн

- Поле ввода — **`<textarea>`** (обёртка shadcn `Textarea`), не однострочный `<input>`.
- Ряд: `flex`, поле слева (`flex-1`), круглая кнопка отправки справа; границы/цвета pill-поля и кнопки — как в текущих классах компонента (не менять без явной задачи на редизайн).

---

## Авто-рост высоты (обязательно через JS)

Одних CSS `height: auto` на `<textarea>` **недостаточно** для предсказуемого поведения: высота задаётся скриптом по `scrollHeight`.

- После измерения: сначала `height = "auto"`, затем `height = min(scrollHeight, MAX_PX)`.
- **Максимум высоты** — константа в коде (сейчас **120px**), что соответствует примерно **5 строкам** при `line-height: 20px` (`leading-5` в Tailwind).
- При превышении лимита — **вертикальный скролл внутри** поля: `max-height`, `overflow-y: auto`, `resize: none`.
- Синхронизация с текстом из React:
  - `useLayoutEffect` при изменении `value` (вставка, очистка после отправки, prefill);
  - `onInput` с той же функцией подстройки — для отзывчивости при наборе.
- Опционально: лёгкая **`transition` по высоте** (~0.1s) для плавности.

---

## Выравнивание кнопки отправки

- Контейнер ряда с полем и кнопкой: **`align-items: flex-end`** (`items-end` в Tailwind), чтобы при многострочном тексте кнопка оставалась у **нижнего** края, а не «плавала» по центру у высокого поля.

---

## Клавиатура

| Действие | Поведение |
|----------|-----------|
| **Enter** | Отправить сообщение (`preventDefault`), если не нужен перенос. |
| **Shift+Enter** | Перенос строки в поле. |

Обработчик находится в **`ChatPage.tsx`** (`handleKeyDown` и передача в `ChatInputBar` как `onKeyDown`), а не внутри `ChatInputBar`.

---

## Ref и фокус

- `ChatInputBar` пробрасывает ref на нативный **`HTMLTextAreaElement`** (callback ref объединяет внутренний ref для измерения высоты и внешний ref из родителя).
- В **`ChatPage`** хранится `textareaRef` для программного фокуса (например после выбора фразы из быстрых подсказок / см. вызовы `focus()` в `ChatPage`).

---

## Что обновлять вместе с кодом

При изменении лимита строк, логики высоты, клавиш или выравнивания — **обновить этот файл в том же changeset**, чтобы описание оставалось источником истины для UI чата.
