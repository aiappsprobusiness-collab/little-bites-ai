# Юридические тексты и согласие при регистрации

## Где лежит контент (source of truth)

| Документ | React-компонент | Маршруты / встраивание |
|----------|-----------------|-------------------------|
| Пользовательское соглашение | `src/components/legal/TermsContent.tsx` | `/terms`, вкладка «Соглашение» в `ProfilePage` |
| Политика конфиденциальности | `src/components/legal/PrivacyContent.tsx` | `/privacy`, вкладка «Конфиденциальность» в `ProfilePage` |
| Условия подписки | `src/components/legal/SubscriptionContent.tsx` | `/subscription/terms`, вкладка «Подписка» в `ProfilePage` |

Страницы-обёртки: `src/pages/legal/Terms.tsx`, `Privacy.tsx`, `Subscription.tsx` — только layout + тот же `*Content`.

## Регистрация

- На вкладке «Начать» в `src/pages/AuthPage.tsx` обязательный чекбокс **без значения по умолчанию** (`acceptLegal: false`), валидация через zod.
- Ссылки ведут на `/terms` и `/privacy` (React Router `Link`).

### Логирование согласия пользователя

При регистрации с отмеченным чекбоксом в Supabase `signUp` в **user metadata** передаётся `accepted_terms_version` (значение из `src/constants/legalVersions.ts` — `LEGAL_TERMS_VERSION`). Триггер **`handle_new_user_v2`** при создании пользователя переносит данные в `public.profiles_v2`:

- **`accepted_terms_at`** — `timestamptz`, момент фиксации на сервере (если версия передана).
- **`accepted_terms_version`** — строка версии юртекстов (соглашение + политика считаются одной версией для учёта).

Источник версии в коде: **`src/constants/legalVersions.ts`**. При существенных правках документов меняют эту константу.

**Повторное согласие** при следующих входах не запрашивается (отдельный UX не реализован). Актуальные условия после обновления текстов определяются пунктом соглашения «Изменение документов» в `TermsContent.tsx`; сохранённая при регистрации версия остаётся записью о моменте первого принятия.

### Дополнительное усиление юридических документов (март 2026)

- добавлена формулировка «как есть» / as is в блок ограничения ответственности (`TermsContent.tsx`);
- добавлено право изменять сервис (функции, интерфейс, тарифы, trial, лимиты) без обещания сохранения прошлого функционала (`TermsContent.tsx`);
- добавлено право изменять юридические документы и связка с продолжением использования сервиса (`TermsContent.tsx`);
- в политику добавлен блок о сроках хранения и удалении/обезличивании данных (`PrivacyContent.tsx`, заголовок раздела 4 уточнён);
- в условия подписки усилён блок возвратов: индивидуальный порядок, критерии рассмотрения, отсутствие гарантии автоматического возврата (`SubscriptionContent.tsx`).

Если после `signUp` у клиента есть **сессия**, дополнительно выполняется best-effort `UPDATE profiles_v2` теми же полями; при ошибке в консоль пишется предупреждение, **регистрация не откатывается**. При регистрации только по email (сессии нет до подтверждения почты) запись полагается на триггер и metadata.

## Продуктовые допущения в юртекстах (март 2026)

- **Автопродление:** в `create-payment` нет recurrent/Rebill; в условиях подписки зафиксировано отсутствие автопродления в текущей версии. При смене модели оплаты обновить `SubscriptionContent` и при необходимости `TermsContent`.
- **Trial:** длительность в коде — `TRIAL_DURATION_DAYS` в `src/utils/subscriptionRules.ts` (3). Тексты согласованы с этим значением.
