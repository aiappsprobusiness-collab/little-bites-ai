# Applied audit: семейная пачка chat_ai (2026-03-21)

Краткий разбор выгрузки рецептов и логов Edge (`deepseek-chat`) после точечного усиления гейта описания (`profile_adaptation_leak` в `sanitizeAndRepair.ts`).

## 1. Description с «без молока» (тушёная баранина)

- **Рецепт:** «Тушёная баранина с овощами и пюре из цветной капусты», `description`: «…кремовым пюре **без молока**…».
- **Почему:** в промпт семьи ушёл блок «ИСКЛЮЧИТЬ (аллергия): белок коровьего молока»; модель отразила адаптацию в user-facing описании.
- **Норма:** для публичного/pool-safe текста — **нежелательно** (раскрывает контекст ограничений семьи).
- **Действие:** серверный гейт `descriptionHasProfileAdaptationLeak` + отказ с причиной **`profile_adaptation_leak`** → `pickCanonicalDescription` подставляет детерминированный benefit-fallback (как и при других отказах гейта). Промпт не менялся.

## 2. `chef_advice = null` (гречневая каша с грибами и яйцом-пашот)

- **Лог:** `CHEF_ADVICE_REJECTED_NULL_AFTER_GUARD`, `rejection_reason`: **`missing_anchor:no_recipe_anchor_no_concrete_cue`**; сырой совет про воронку для яйца-пашот был, после пайплайна low-value gate обнулил.
- **Оценка:** **корректное** поведение quality gate: совет был в основном про технику яйца, слабо привязан к якорям рецепта (гречка/грибы) в эвристике `isChefAdviceLowValue`.
- **Действие:** автоматически не меняли; при желании позже — узкий тюнинг якорей для «яйцо-пашот» в составе блюда (отдельная задача, вне этого фикса).

## 3. `canonicalDescriptionSource = deterministic_fallback`

- **Примеры из логов:** омлет с авокадо, стейк тунца, тыквенный суп с чечевицей — `DESCRIPTION_QUALITY_GATE_RAW_LLM` / `descOk: false`, затем `RECIPE_SANITIZED` с `canonicalDescriptionSource: deterministic_fallback`.
- **Почему:** сырой LLM `description` не прошёл `passesDescriptionQualityGate` (длина/два предложения/штампы/anchoring и т.д. — по логам без единой причины на все рецепты).
- **Утечки адаптации в fallback:** шаблоны `buildDescriptionFallback` / benefit-пулы **не** содержат формулировок «без молока»/аллергии; регрессии закрыты тестом «buildDescriptionFallback … не содержит leak».
- **Норма:** **ожидаемое** использование канала fallback; после фикса новые утечки профиля в финальный текст через LLM-description режутся, fallback остаётся нейтральным.

## Риски ложных срабатываний

- Паттерны нацелены на типичные аллергены и явные фразы адаптации; **не** матчят нейтральные «без майонеза», «без лишней тяжести», «на молоке» в названии блюда.
- Возможны редкие коллизии: формулировки вроде «без ореховой пасты» (кульинарный смысл) могут попасть под `\bбез\s+орех`, если текст начнётся с «без орехов…» — при появлении в проде смотреть по логам `rejection_reason`.
