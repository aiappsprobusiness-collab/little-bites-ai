# Рефактор UX списка покупок (март 2026)

## Цель

Уйти от ощущения «скрытого зеркала плана» к модели **явной сборки снимка** списка из меню, с основным входом с экрана **План** и спокойной коммуникацией при расхождении с планом.

## Сделано

- CTA **«Собрать список продуктов»** на `MealPlanPage` после списка блюд дня → sheet **BuildShoppingListFromPlanSheet** → после сборки навигация на список. Ниже — ghost **«Отправить меню»** (если есть блюда): модалка превью «Ваше меню» → **«Отправить»** → системный share или копирование; текст и ссылка — `src/utils/shareMenuText.ts`. Верх блока плана: «Собрать день» / «Собрать неделю».
- В Избранном: убран третий равный чип «Список продуктов»; добавлена вторичная ссылка **«Покупки»** и **«К избранному и рецептам»** на экране списка. (Март 2026: «Покупки» вынесены в отдельную спокойную строку под сегментом «Избранное / Мои рецепты» — см. **docs/dev/favorites-tab-ui-quiet-2026-03.md**.)
- Удалён баннер «Меню изменилось» / «Обновить список»; вместо него при расхождении — нейтральная полоса и **«Собрать заново»**.
- Вынесены `loadPlanShoppingIngredients` и `loadPlanSignature` для переиспользования вне React Query.
- Согласование `member_id` с планом: `mealPlanMemberIdForShoppingSync` (в т.ч. Free + семья).

### Итерация UI (блок плана + список + фильтр)

- План: под приёмами пищи — «Собрать список продуктов», затем «Отправить меню»; в шапке — «Собрать день», «Собрать неделю».
- Список покупок: короткий заголовок по периоду; копирование под списком и в меню «Ещё»; фильтр ghost; упрощённая модалка фильтра.
