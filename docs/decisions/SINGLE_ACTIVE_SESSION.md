# Одна активная сессия на аккаунт

**Решение:** один аккаунт = одна активная сессия / одно активное устройство. При входе с другого устройства предыдущее теряет доступ.

## Реализация

- **Подход:** `active_session_key` в `public.profiles_v2`.
- При каждом новом логине (password или magic link / auth callback) генерируется новый случайный ключ, сохраняется в БД и в `localStorage` на клиенте.
- Проверка валидности сессии:
  - при старте приложения (после `getSession`);
  - при смене сессии (`onAuthStateChange`);
  - при возврате в приложение (`visibilitychange`).
- Если ключ в БД не совпадает с ключом в `localStorage` (или локального ключа нет при наличии ключа в БД), сессия считается недействительной: выполняется `signOut`, очищается локальный ключ, в `sessionStorage` записывается причина. При переходе на `/auth` пользователю показывается сообщение: «Аккаунт открыт на другом устройстве. Войдите снова.»

## Где обновляется активная сессия

- **Логин по паролю:** `useAuth.signIn` после успешного `signInWithPassword` вызывает `setActiveSessionKeyForUser(userId)`.
- **Magic link / email confirmation:** `AuthCallbackPage` после получения сессии вызывает `setActiveSessionKeyForUser(userId)` перед редиректом.

## Где выполняется проверка

- `useAuth`: эффект инициализации (getSession + проверка), подписка на `onAuthStateChange`, обработчик `visibilitychange`. При невалидной сессии: `setSessionInvalidReason`, `clearStoredSessionKey`, `supabase.auth.signOut()`.

## UX

- Сообщение при принудительном выходе: **«Аккаунт открыт на другом устройстве. Войдите снова.»**
- Показ через toast на странице `/auth` при монтировании, если в `sessionStorage` сохранена причина `replaced`.

## БД

- Колонка `profiles_v2.active_session_key` (text, nullable). Миграция: `20260312180000_profiles_v2_active_session_key.sql`.

## Безопасность

- Источник истины — сервер (значение в БД). Клиент только сравнивает свой ключ с ключом из БД. Подмена только локального ключа не даёт доступа, т.к. при следующем запросе к БД будет использоваться актуальный ключ профиля.
