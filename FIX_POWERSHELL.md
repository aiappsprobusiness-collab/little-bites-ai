# Решение проблемы с PowerShell Execution Policy

## Проблема
PowerShell блокирует выполнение скриптов npm из-за политики безопасности.

## Решение 1: Изменить политику выполнения (рекомендуется)

Откройте PowerShell **от имени администратора** и выполните:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

Нажмите `Y` для подтверждения.

После этого перезапустите терминал и попробуйте снова:
```bash
npm run dev
```

## Решение 2: Использовать командную строку (cmd)

Вместо PowerShell используйте обычную командную строку Windows:

1. Нажмите `Win + R`
2. Введите `cmd` и нажмите Enter
3. Перейдите в папку проекта:
   ```cmd
   cd "C:\Users\alesa\OneDrive\Рабочий стол\Repositories\little-bites-ai"
   ```
4. Выполните команды:
   ```cmd
   npm run dev
   ```

## Решение 3: Временно обойти политику

В текущей сессии PowerShell выполните:

```powershell
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process
```

Это изменит политику только для текущей сессии PowerShell.

## Решение 4: Использовать Git Bash

Если у вас установлен Git, используйте Git Bash:
1. Откройте Git Bash
2. Перейдите в папку проекта
3. Выполните команды npm как обычно

## Проверка текущей политики

Чтобы проверить текущую политику выполнения:
```powershell
Get-ExecutionPolicy -List
```

## Дополнительная информация

- `RemoteSigned` - позволяет выполнять локальные скрипты и загруженные скрипты с цифровой подписью
- `CurrentUser` - применяется только к текущему пользователю (безопаснее, чем для всей системы)
- После изменения политики npm команды должны работать нормально
