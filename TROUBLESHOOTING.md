# Решение проблем с запуском приложения

## Проблема: Приложение не запускается

### 1. Проверьте ошибки в консоли браузера

Откройте DevTools (F12) и проверьте вкладку Console на наличие ошибок.

### 2. Проверьте файл .env

Убедитесь, что файл `.env` создан в корне проекта со следующим содержимым:

```env
VITE_GIGACHAT_CLIENT_SECRET_KEY=MDE5YmQyOGItMzU3OC03MWE4LWFhZGQtNTM1NzUzZjFkYjEzOjZhNWI2MmJiLTkxMTgtNDFjOS1hOWEyLTIwNmMzMWYxMzY0ZQ==
VITE_GIGACHAT_IS_PERSONAL=true
```

### 3. Перезапустите dev сервер

После создания/изменения `.env` файла **обязательно перезапустите** dev сервер:

1. Остановите сервер: `Ctrl+C`
2. Запустите снова: `npm run dev`

### 4. Очистите кеш

Если проблемы продолжаются, очистите кеш:

```bash
# Удалите папку node_modules/.vite (если есть)
rm -rf node_modules/.vite

# Или в Windows:
rmdir /s node_modules\.vite

# Перезапустите сервер
npm run dev
```

### 5. Проверьте установку зависимостей

Убедитесь, что все зависимости установлены:

```bash
npm install
```

### 6. Проверьте порт 8080

Если порт 8080 занят, измените его в `vite.config.ts`:

```typescript
server: {
  port: 3000, // или другой свободный порт
}
```

### 7. Проблемы с GigaChat

Если приложение не запускается из-за GigaChat:

- Приложение должно работать даже без GigaChat
- Если есть ошибки, проверьте формат ключа в `.env`
- Убедитесь, что ключ в Base64 формате

### 8. Проверьте логи терминала

Проверьте вывод в терминале на наличие ошибок компиляции TypeScript или других проблем.

## Частые ошибки

### "Cannot find module"
```bash
npm install
```

### "Port already in use"
Измените порт в `vite.config.ts` или остановите процесс, использующий порт 8080.

### "GigaChat не настроен"
Это предупреждение, не ошибка. Приложение должно работать, просто без AI функций.

## Если ничего не помогает

1. Удалите `node_modules` и `package-lock.json`
2. Выполните `npm install` заново
3. Убедитесь, что Node.js версии 18+
4. Проверьте, что файл `.env` в корне проекта (рядом с `package.json`)
